# Phigmund Demo Emails — Pre-Selected Test Cases

Five terrible IT emails, ready to paste into `email_responder.py`. Ranked from "warm up" to "final boss."

Run the tool with:
```bash
python phigmund/email_responder.py --save
```
Paste the email text when prompted. Press ENTER twice when done.

---

## Demo Email #1 — The Classic (Start Here)
*Category: Slow computer + unreasonable deadline + implicit blame*

```
From: Karen Miller <kmiller@company.com>
To: IT Help Desk
Subject: URGENT - computer is making noise and running slow PLEASE HELP

Hi,

My computer has been running really slow for like 3 weeks now and it's been making a clicking noise sometimes. Also I think I need admin rights to install something for a meeting I have at 2pm TODAY. The software is called DownloadAllFreeStuff.exe and my friend emailed it to me as an attachment.

Also sometimes my Outlook freezes but it goes away if I wait. And my printer isn't working but it was working last week. I might have spilled something near it but I don't think that matters.

I need all of this fixed before 2pm. I have a very important meeting and this is really affecting my productivity.

Thanks,
Karen
```

**Why this one works for the demo:** It hits every classic — urgency without emergency, three weeks of waiting until a 2-hour deadline, installing executable from email attachment, and the legendary "I don't think that matters" next to the confession. Phigmund will have a lot to work with here.

## Demo Email #2 — The "Quick Ask"
*Category: Massive scope disguised as a tiny favor*

```
From: Dave Callahan <dcallahan@company.com>
To: sysadmin@company.com
Subject: quick ask - shouldn't take long

Hey,

Quick ask - shouldn't take long.

I need you to set up a new SharePoint site for our project, migrate all the files from the old SharePoint site we had in 2019 (I don't remember the URL but you probably have it), set up permissions for the new team (I'll send you the list - still building it), and create an automated workflow that notifies people when documents are updated. 

Oh also can you make sure the old files are still accessible from the old location? We might need to reference them.

Also we need this by end of day Friday because we're presenting to the board Monday and I told them this would be done.

Shouldn't take long for someone who knows what they're doing.

Thanks
Dave

P.S. I cc'd my VP just so she's in the loop
```

**Why this one works:** "Shouldn't take long for someone who knows what they're doing" is the kill shot. Plus VP cc (pressure). Plus Friday deadline for a Monday board meeting. Plus "I'll send you the team list - still building it." Phigmund's assessment will be devastating and accurate.

## Demo Email #3 — The Password Policy Philosopher
*Category: Security policy resistance with philosophical framing*

```
From: Robert "Bob" Hendricks <rhendricks@company.com>
To: IT Security
Subject: RE: RE: RE: RE: Password expiration notice - FINAL WARNING

Look,

I've been using the same password for 14 years and I have NEVER been hacked. This new policy requiring me to change my password every 90 days is making it impossible to do my job. 

I had to write my last 4 passwords on a sticky note on my monitor because I can't remember them. My current password strategy is to add a number at the end and increase it by one each time, which I think is just as secure.

I don't understand why IT gets to make these decisions when we're the ones actually doing the work around here. I went over your head and talked to Linda in HR and she said she "didn't know anything about a password policy" which I think speaks for itself.

Can you just make my account exempt? I've been here longer than you have and I promise I'll be careful.

Bob

(14 years, not a single hack, that's my track record)
```

**Why this one works:** Every IT professional has Bob. The sticky note confession, the "I went over your head to HR," the seniority argument, and the proud declaration of an obviously compromised password strategy. Phigmund's "merely observational" response to the sticky note alone is worth recording.

## Demo Email #4 — The Escalation Artist
*Category: Manufactured urgency + implicit threat*

```
From: Patricia Vance <pvance@company.com>
To: IT Help Desk; CIO@company.com; mymanager@company.com; itsupervision@company.com
Subject: ESCALATION - Printer Issue - Day 7 - Still No Resolution - Unacceptable

To Whom It May Concern,

This is my seventh email on this matter. I have attached the previous six emails for reference (attachments 1-6). I have also attached a screenshot of my original ticket, a photo of the printer, a photo of the error message, a photo of the room the printer is in so you can locate it, and a Word document I wrote summarizing the situation.

The printer on Floor 3, West Wing, near the window (it is the one with the yellow Post-It note on it that says "broken - do not use" - I put that note there) continues to not print my documents. My documents are very important. 

I would like to speak to a supervisor. I would also like a written explanation for why this has taken 7 days. I have copied my manager, the CIO, and what I believe is the IT supervision email address (I found it on an old company directory from 2018, I hope it is still correct).

This is affecting my productivity and possibly my health.

Patricia Vance
Senior Associate (13 years)

(Six attachments included: previous_emails_1-6.zip, printer_photo.jpg, error_message.jpg, room_photo.jpg, summary_document_v3_FINAL_updated.docx)
```

**Why this one works:** The yellow Post-It note detail is perfect (she broke it herself). CIO cc. Found a supervision email from a 2018 directory. Word document summarizing the situation. "Possibly my health." Phigmund will note the number of attachments and check if any of them are 42.

```

Subject: server

hey i think i deleted something on the server

not sure what but something looked different when i was on there earlier. also i may have been on the wrong server? theres multiple ones right? i was just copying some files but i might have been doing something else also

anyway probably fine but wanted to let you know

going on vacation for 2 weeks starting in like an hour

-Mark

```

**Why this one works:** "I think I deleted something. Not sure what." The grammatical minimalism. Multiple servers. Going on vacation in an hour. "Probably fine." Sent from phone. Phigmund's assessment of this email may contain his most cosmically accurate observation to date.

## Recording Notes

When you record the demo video, the order that works best for an audience:

1. **Start with #1 (Karen)** — relatable warmup, everyone has a Karen email
2. **Do #5 (Mark on vacation)** — big finale energy, the "probably fine" reveal lands perfectly
3. **Save #4 (Patricia escalation)** for the second video — it's its own full episode

The comedy timing: Read the input email slowly and dramatically. Let Phigmund's response appear (or read it out loud). The pause before reading "THE RESPONSE YOU WANTED TO SEND" is where the setup lands.

Don't explain the format before running the tool. Let Phigmund explain himself.
