# Video Distribution Checklist

Triggers on: "ready to post this video," "let's distribute this," "post the
Jarvis video," "where do I need to post this," or any similar signal that a
finished video is about to go out. Walk Randy through this list in order,
checking off each platform as confirmed — don't assume any step happened just
because a video exists. This is a manual-process memory aid, not automation:
the point is nothing gets forgotten after a long build-and-edit day, not to
replace Randy actually doing the posting.

**Scope note:** this is specific to the current build-in-public / Jarvis
series manual workflow (2026-07-17). Randy has explicitly chosen NOT to
automate this yet — weeks spent perfecting a posting pipeline is time content
isn't sitting out there being discovered. Keep this checklist lightweight;
don't propose automation unless Randy asks.

## The checklist, in order

1. **YouTube** — main video upload. Title, description, thumbnail.
2. **LinkedIn — own feed** — native video post.
3. **LinkedIn — target group(s)** — post in the one AI/IT-related group Randy
   posts to (get the exact group name from Randy if not already known this
   session — don't guess it). **Known gotcha:** native video upload to LinkedIn
   groups sometimes silently fails. If it doesn't take, don't just retry blind
   — flag it to Randy so he can check whether the group needs a different
   format (e.g., a link post instead of native video).
4. **Facebook — personal page** — native video post, only if the video fits
   the page's tone.
5. **Facebook — groups (4-5 of them)** — **native upload only, never "share."**
   Always fill in the description field — this materially lowers the odds of
   a group admin rejecting the post. Get the exact group list from Randy if
   not already known this session.
6. **Jared's Discord — "win for today" channel** — **text only, no images or
   video natively supported in this channel.** Every post here must include:
   - The actual update/win, written as text
   - The GitHub repo link, or the prompt(s) pasted directly — Randy commits to
     giving away the repo/prompt on every video, so this is a promise to keep,
     not optional
   - **Known unresolved item, not part of the checklist yet:** Randy activated
     a YouTube app integration in his Discord profile to try to share a
     playlist visually (build-in-public footage) since the channel is
     text-only otherwise. It's not working yet — possibly disabled by Jared
     server-side. Don't treat this as available until Randy confirms it works.

## What this skill does NOT do

- Does not post anything automatically — Randy does every step by hand.
- Does not track which specific videos have been posted where (that's a
  separate, future problem if the manual load becomes unsustainable — not
  this skill's job).
- Does not expand to future platforms until Randy explicitly says so (he's
  stated he wants to expand distribution later, once the current manual
  approach proves out — don't get ahead of that).
