## Phase 3 Scope — Build exactly these things

### 1. Pain Point Extractor (src/analyzer/pain_point_extractor.py)

This is the core business intelligence module. It scans all transcript and
comment files in a specified category group and uses the Claude API to identify
and rank the most repeated audience questions, pain points, and desired outcomes.

**Two input sources per video (weighted differently):**
- Transcript .md files — what the creator says (lower weight)
- Comment _comments.md files — what the audience asks and struggles with
  (higher weight — 2x signal value for pain point research)

**Processing approach (token-efficient):**

Do NOT send all transcripts to Claude in one call — context window will overflow.
Use a two-pass chunked approach:

Pass 1 — Per-file extraction:
For each transcript/comment file pair, make one Claude API call that extracts:
- Top 5 questions asked or implied in this content
- Top 5 pain points expressed
- Top 3 desired outcomes mentioned
Return structured JSON only — no prose.

Pass 2 — Aggregation:
After all files are processed, make one final Claude API call that receives
all per-file JSON results and aggregates them into a master ranked list with
mention frequency counts.

**Claude API call spec for Pass 1:**
```python
{
    "model": "claude-haiku-4-5-20251001",  # cheapest model — sufficient for extraction
    "max_tokens": 500,
    "system": "You are a market research analyst. Extract pain points and questions from content. Return ONLY valid JSON. No prose. No markdown.",
    "messages": [
        {
            "role": "user",
            "content": f"""Analyze this content and return JSON with exactly this structure:
{{
  "questions": ["question 1", "question 2", "question 3", "question 4", "question 5"],
  "pain_points": ["pain 1", "pain 2", "pain 3", "pain 4", "pain 5"],
  "desired_outcomes": ["outcome 1", "outcome 2", "outcome 3"]
}}

TRANSCRIPT CONTENT:
{transcript_chunk}

COMMENT CONTENT (weight 2x):
{comments_chunk}"""
        }
    ]
}
```

**Claude API call spec for Pass 2 (aggregation):**
```python
{
    "model": "claude-haiku-4-5-20251001",
    "max_tokens": 2000,
    "system": "You are a market research analyst. Aggregate and rank pain point data. Return ONLY valid JSON. No prose.",
    "messages": [
        {
            "role": "user",
            "content": f"""Aggregate these per-video extractions into a master ranked list.
Count how many videos mention each theme. Merge similar items.
Return JSON with exactly this structure:
{{
  "top_questions": [
    {{"text": "question text", "count": 5, "category": "getting started"}},
    ...
  ],
  "top_pain_points": [
    {{"text": "pain point", "count": 8, "category": "technical"}},
    ...
  ],
  "top_desired_outcomes": [
    {{"text": "outcome", "count": 6, "category": "career"}},
    ...
  ],
  "videos_analyzed": 20,
  "total_extractions": 60
}}

DATA:
{json.dumps(all_extractions)}"""
        }
    ]
}
```

### 2. Knowledge Base Indexer (src/knowledge_base/indexer.py)

Build a lightweight index of all transcript files without requiring a vector
database. Plain JSON index — sufficient for Phase 3.

**Index structure (knowledge_base/index.json):**
```json
{
  "built_at": "ISO timestamp",
  "total_files": 0,
  "groups": {
    "ai-and-claude-code": {
      "count": 20,
      "files": [
        {
          "filename": "2026-04-12_video-title.md",
          "path": "transcripts/ai-and-claude-code/filename.md",
          "title": "Video Title",
          "channel": "Channel Name",
          "published": "YYYY-MM-DD",
          "word_count": 5432,
          "has_comments": true,
          "comments_path": "transcripts/ai-and-claude-code/filename_comments.md"
        }
      ]
    }
  }
}
```

**CLI command:**
```bash
python -m src.main index
```

Always rebuild index from scratch — never append. Index is derived data,
transcripts are the source of truth.

### 3. On-Demand Q&A (src/knowledge_base/query.py)

Build a Q&A module that searches the transcript library to answer questions.

**Approach:**
- Load index.json to find relevant files
- Simple keyword search to identify top 5 most relevant transcripts
- Send those 5 transcripts (chunked if needed) to Claude API
- Claude answers based only on transcript content — not general knowledge

**Tool count rule:** Maximum 1 Claude API call per query in Phase 3.
If the answer requires more context, tell the user which files to read directly.

**CLI command:**
```bash
python -m src.main ask "What are the best AI certifications for career changers?"
python -m src.main ask --group ai-and-claude-code "What tools do experts recommend?"
```

**Output format:**
```
ANSWER (based on 5 transcripts from ai-and-claude-code):

[Claude's answer here]

SOURCES:
- 2026-04-12_video-title.md (AI News & Strategy Daily)
- 2026-04-10_another-video.md (Channel Name)
...
```

**Rule:** If no relevant transcripts found, say so clearly. Never hallucinate
an answer from general knowledge. Always cite sources.
